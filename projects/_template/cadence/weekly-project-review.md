# Weekly Project Review

Use this once per week or at the end of a delivery cycle.

## Prompt

```text
Using this project workspace, run a weekly project review. Summarize progress, risks, decisions, dependency movement, stakeholder updates needed, and the recommended focus for next week. Ground every claim in the project files and call out missing or stale context.
```

## Review areas

- Progress against the work plan
- Current status and trend
- Risks, assumptions, issues, and dependencies
- Decisions made or needed
- Stakeholder communication needs
- Next week's focus
