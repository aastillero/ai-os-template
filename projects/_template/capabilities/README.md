# Project Capabilities

Capabilities are the repeatable ways this project gets work done.

Start with simple documented workflows and prompts. Only turn a workflow into a Copilot skill after it is proven, repeated, and the user explicitly approves adding skills.

## Capability layers

1. Manual workflow — documented in `workflows.md`.
2. Reusable prompt — documented in `prompts.md`.
3. Candidate skill — tracked in `skill-backlog.md`.
4. Copilot skill — later added under `.github/skills/<skill-name>/SKILL.md` after approval.
